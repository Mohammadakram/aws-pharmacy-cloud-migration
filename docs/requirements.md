# Requirements

## Functional Requirements

- Store patient prescription data and a shared medicine inventory for all four branches.
- Update inventory in real time when a sale is made.
- Generate daily sales reports and low-stock alerts for medicines.
- Provide secure logins for authorised pharmacy staff.

## Non-Functional Requirements

- Performance: The system must be responsive with low latency for daily pharmacy use.
- Security: Only authorised staff can access prescription data, and data privacy must meet healthcare standards.
- Technical constraints: The new system should run on existing pharmacy computers without new hardware.
- Project constraints: Implementation and testing must be completed within eight weeks.
- Usability: The interface must be simple and easy for staff to learn.
